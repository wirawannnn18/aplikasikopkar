## Modul Training

### Modul 1: Pengenalan Interface (30 menit)

#### 1.1 Login dan Dashboard
**Praktik:**
1. Buka browser dan akses sistem POS
2. Login dengan username dan password yang diberikan
3. Kenali menu utama: Dashboard, POS, Keuangan, Laporan

**Yang Perlu Diperhatikan:**
- Jangan gunakan private/incognito mode
- Bookmark halaman untuk akses cepat
- Pastikan koneksi internet stabil

#### 1.2 Menu Keuangan
**Navigasi:**
1. Klik menu "Keuangan"
2. Lihat submenu: Buka Kas, Tutup Kas, Riwayat Tutup Kasir
3. Pahami fungsi masing-masing menu

**Demo:**
- Supervisor menunjukkan setiap menu
- Kasir mengikuti navigasi
- Tanya jawab tentang fungsi menu

#### 1.3 Halaman POS
**Fitur Utama:**
1. Area transaksi (input barang, hitung total)
2. Header dengan informasi kasir dan shift
3. Tombol "Tutup Kasir" (warna kuning)
4. Mode fullscreen untuk fokus kerja

**Tips Interface:**
- Gunakan F11 untuk fullscreen
- Perhatikan informasi shift di header
- Tombol tutup kasir hanya muncul setelah buka kas

### Modul 2: Proses Buka Kas (45 menit)

#### 2.1 Persiapan Shift
**Sebelum Buka Kas:**
1. Terima modal awal dari supervisor
2. Hitung dan verifikasi jumlah modal
3. Catat jumlah modal untuk input ke sistem

**Checklist Persiapan:**
- [ ] Modal awal sudah diterima dan dihitung
- [ ] Kasir sudah login ke sistem
- [ ] Printer sudah siap dan ada kertas
- [ ] Area kerja sudah bersih dan tertata

#### 2.2 Input Buka Kas
**Langkah-langkah:**
1. Masuk ke menu "Keuangan" â†’ "Buka Kas"
2. Input modal awal (angka saja, tanpa titik/koma)
3. Klik "Buka Kas"
4. Sistem akan generate ID shift dan menyimpan data

**Contoh Input:**
- Modal Rp 500.000 â†’ Input: 500000
- Modal Rp 1.000.000 â†’ Input: 1000000

**Verifikasi:**
- Cek header POS menampilkan info shift
- Tombol "Tutup Kasir" sudah muncul
- ID shift tercatat di sistem

#### 2.3 Praktik Buka Kas
**Exercise 1:**
1. Kasir baru praktik buka kas dengan modal dummy
2. Supervisor mengawasi dan memberikan feedback
3. Ulangi sampai lancar (target: <2 menit)

### Modul 3: Transaksi Normal (60 menit)

#### 3.1 Jenis Transaksi
**Cash vs Kredit:**
- **Cash**: Pembayaran tunai, mempengaruhi kas fisik
- **Kredit**: Pembayaran non-tunai, tidak mempengaruhi kas fisik

**Impact ke Tutup Kasir:**
- Hanya transaksi CASH yang mempengaruhi perhitungan kas
- Sistem otomatis memisahkan kedua jenis transaksi

#### 3.2 Input Transaksi
**Demo Transaksi:**
1. Input barang dan hitung total
2. Pilih metode pembayaran (Cash/Kredit)
3. Input jumlah bayar dan hitung kembalian
4. Print struk untuk pelanggan

**Yang Perlu Diperhatikan:**
- Pastikan metode pembayaran benar
- Hitung kembalian dengan teliti
- Simpan struk untuk rekonsiliasi

#### 3.3 Praktik Transaksi
**Exercise 2:**
1. Simulasi 10 transaksi (5 cash, 5 kredit)
2. Variasi nominal untuk latihan hitung kembalian
3. Supervisor cek akurasi input dan kembalian

### Modul 4: Proses Tutup Kasir (90 menit)

#### 4.1 Persiapan Tutup Kasir
**Sebelum Tutup Kasir:**
1. Pastikan semua transaksi sudah diinput
2. Tidak ada pelanggan yang menunggu
3. Siapkan kalkulator untuk hitung kas fisik
4. Pisahkan uang per pecahan untuk memudahkan hitung

**Checklist Persiapan:**
- [ ] Semua transaksi sudah complete
- [ ] Kas fisik siap dihitung
- [ ] Kalkulator tersedia
- [ ] Struk transaksi tersimpan rapi

#### 4.2 Menghitung Kas Fisik
**Teknik Menghitung:**
1. **Pisahkan per pecahan:**
   - Uang kertas: 100rb, 50rb, 20rb, 10rb, 5rb, 2rb, 1rb
   - Uang logam: 1000, 500, 200, 100, 50

2. **Hitung per pecahan:**
   - Contoh: 5 lembar 100rb = 500.000
   - Catat di kertas untuk verifikasi

3. **Total semua:**
   - Jumlahkan semua pecahan
   - Double-check dengan kalkulator

**Tips Akurasi:**
- Hitung dua kali untuk memastikan
- Minta bantuan rekan jika perlu
- Jangan terburu-buru
- Cek laci bawah dan tempat tersembunyi

#### 4.3 Menggunakan Modal Tutup Kasir
**Membuka Modal:**
- **Cara 1:** Klik tombol "Tutup Kasir" di header POS
- **Cara 2:** Tekan F10 (keyboard shortcut)

**Memahami Informasi Modal:**
1. **Informasi Shift:**
   - Nama kasir dan waktu buka
   - Modal awal yang diinput

2. **Ringkasan Penjualan:**
   - Total penjualan (cash + kredit)
   - Penjualan cash (yang mempengaruhi kas)
   - Penjualan kredit (tidak mempengaruhi kas)
   - Jumlah transaksi

3. **Perhitungan Kas:**
   - Kas seharusnya = Modal awal + Penjualan cash
   - Field input kas aktual
   - Selisih (dihitung otomatis)

#### 4.4 Input Kas Aktual
**Langkah Input:**
1. Masukkan hasil hitung kas fisik di field "Kas Aktual"
2. Sistem otomatis hitung selisih
3. Perhatikan warna indikator selisih:
   - ğŸŸ¢ Hijau: Kas sesuai (selisih = 0)
   - ğŸŸ¡ Kuning: Kas lebih (selisih > 0)
   - ğŸ”´ Merah: Kas kurang (selisih < 0)

**Contoh:**
- Kas seharusnya: Rp 2.500.000
- Kas aktual: Rp 2.495.000
- Selisih: -Rp 5.000 (kurang)

#### 4.5 Keterangan Selisih
**Kapan Perlu Keterangan:**
- Jika selisih â‰  0 (lebih atau kurang)
- Field keterangan muncul otomatis
- WAJIB diisi dengan penjelasan yang jelas

**Contoh Keterangan yang Baik:**
- âœ… "Salah kembalian ke pelanggan Rp 10.000"
- âœ… "Uang pecahan Rp 5.000 sobek, tidak diterima pelanggan"
- âœ… "Lupa input transaksi cash Rp 50.000"

**Contoh Keterangan yang Kurang Baik:**
- âŒ "Tidak tahu"
- âŒ "Lupa"
- âŒ "Salah hitung"

#### 4.6 Submit dan Print
**Proses Submit:**
1. Pastikan semua data sudah benar
2. Klik "Proses Tutup Kasir"
3. Sistem akan:
   - Menyimpan data tutup kasir
   - Membuat jurnal akuntansi (jika ada selisih)
   - Membersihkan data shift
   - Membuka window print

**Print Laporan:**
1. Window print terbuka otomatis
2. Preview laporan sebelum print
3. Klik "Print" atau Ctrl+P
4. Jika print gagal, gunakan alternatif (PDF/Email)

#### 4.7 Praktik Tutup Kasir
**Exercise 3:**
1. Simulasi tutup kasir dengan data dummy
2. Praktik hitung kas fisik
3. Input ke sistem dan print laporan
4. Ulangi dengan berbagai skenario selisih

### Modul 5: Serah Terima dengan Supervisor (30 menit)

#### 5.1 Prosedur Serah Terima
**Yang Diserahkan:**
1. Laporan tutup kasir (print)
2. Kas fisik (sesuai kas aktual)
3. Struk transaksi (jika diperlukan)
4. Penjelasan lisan jika ada selisih

#### 5.2 Verifikasi Supervisor
**Yang Diperiksa Supervisor:**
1. Kelengkapan laporan
2. Kesesuaian kas fisik dengan laporan
3. Keterangan selisih (jika ada)
4. Tanda tangan di laporan

#### 5.3 Dokumentasi
**Arsip yang Disimpan:**
- Copy laporan tutup kasir
- Catatan selisih (jika ada)
- Tanda tangan supervisor
- Waktu serah terima

## Hands-On Practice

### Sesi Praktik 1: Simulasi Shift Lengkap (2 jam)

#### Setup Simulasi
**Data Dummy:**
- Modal awal: Rp 1.000.000
- 20 transaksi (15 cash, 5 kredit)
- Total penjualan cash: Rp 2.500.000
- Kas seharusnya: Rp 3.500.000

#### Skenario A: Kas Sesuai
**Target Learning:**
- Proses normal tanpa selisih
- Fokus pada akurasi dan kecepatan

**Steps:**
1. Buka kas dengan modal Rp 1.000.000
2. Input 20 transaksi sesuai data
3. Hitung kas fisik = Rp 3.500.000
4. Tutup kasir (selisih = 0)
5. Print laporan dan serah terima

**Success Criteria:**
- Selesai dalam 45 menit
- Tidak ada error input
- Laporan print berhasil

#### Skenario B: Kas Kurang
**Target Learning:**
- Handling selisih negatif
- Membuat keterangan yang baik

**Steps:**
1. Sama seperti Skenario A
2. Kas fisik sengaja dikurangi Rp 10.000
3. Input kas aktual Rp 3.490.000
4. Isi keterangan: "Salah kembalian ke pelanggan Rp 10.000"
5. Submit dan print

**Success Criteria:**
- Keterangan jelas dan spesifik
- Tidak panik saat ada selisih
- Prosedur tetap diikuti dengan benar

#### Skenario C: Kas Lebih
**Target Learning:**
- Handling selisih positif
- Memahami penyebab kas lebih

**Steps:**
1. Sama seperti Skenario A
2. Kas fisik ditambah Rp 5.000
3. Input kas aktual Rp 3.505.000
4. Isi keterangan: "Pelanggan lupa ambil kembalian Rp 5.000"
5. Submit dan print

**Success Criteria:**
- Memahami berbagai penyebab kas lebih
- Keterangan akurat dan detail

### Sesi Praktik 2: Error Handling (1 jam)

#### Skenario Error Umum
**Error 1: Session Hilang**
- Simulasi: Hapus sessionStorage
- Learning: Recovery procedure
- Practice: Buka kas ulang

**Error 2: Print Gagal**
- Simulasi: Block popup browser
- Learning: Alternative print methods
- Practice: Download PDF, email report

**Error 3: Input Validation**
- Simulasi: Input kas aktual negatif
- Learning: Validation rules
- Practice: Correct input format

#### Troubleshooting Practice
**Exercise 4:**
1. Kasir menghadapi error sengaja dibuat
2. Gunakan troubleshooting guide
3. Resolve error dengan minimal bantuan
4. Document lesson learned

### Sesi Praktik 3: Speed & Accuracy (1 jam)

#### Time Trial
**Target Metrics:**
- Buka kas: <2 menit
- Input 10 transaksi: <15 menit
- Tutup kasir: <10 menit
- Total shift: <30 menit

#### Accuracy Test
**Zero Error Target:**
- Semua input harus akurat
- Perhitungan kas harus tepat
- Keterangan harus jelas (jika ada selisih)

#### Efficiency Tips
**Keyboard Shortcuts:**
- F10 untuk buka modal tutup kasir
- Tab untuk navigasi field
- Enter untuk submit

**Best Practices:**
- Siapkan keterangan standar
- Hitung kas per pecahan
- Double-check sebelum submit

## Assessment & Certification

### Assessment Criteria

#### Knowledge Test (25%)
**Multiple Choice Questions (20 soal):**
1. Kapan tombol tutup kasir muncul?
2. Apa yang mempengaruhi kas seharusnya?
3. Kapan keterangan selisih wajib diisi?
4. Bagaimana cara recovery jika session hilang?
5. Apa yang harus dilakukan jika print gagal?

**Passing Score:** 80% (16/20 benar)

#### Practical Test (50%)
**Simulasi Shift Lengkap:**
1. Buka kas dengan modal yang diberikan
2. Input 15 transaksi (mix cash/kredit)
3. Tutup kasir dengan kas aktual yang ditentukan
4. Handle selisih jika ada
5. Print laporan dan serah terima

**Assessment Points:**
- Akurasi input (20 poin)
- Kecepatan eksekusi (15 poin)
- Error handling (10 poin)
- Keterangan selisih (5 poin)

**Passing Score:** 40/50 poin

#### Troubleshooting Test (25%)
**Error Scenarios:**
1. Session corrupted - recovery procedure
2. Print failure - alternative methods
3. Validation error - correct input
4. Data inconsistency - verification steps

**Assessment:**
- Problem identification (10 poin)
- Solution implementation (10 poin)
- Documentation (5 poin)

**Passing Score:** 20/25 poin

### Certification Levels

#### Level 1: Basic Operator
**Requirements:**
- Pass all assessments
- Complete 40 hours supervised practice
- Zero critical errors in last 10 shifts

**Privileges:**
- Operate POS independently
- Perform tutup kasir with supervisor approval
- Access basic reports

#### Level 2: Advanced Operator
**Requirements:**
- 6 months experience as Level 1
- Advanced troubleshooting certification
- Train new kasir (2 successful trainees)

**Privileges:**
- Perform tutup kasir independently
- Handle complex error scenarios
- Access advanced reports
- Assist in training new kasir

#### Level 3: Supervisor
**Requirements:**
- 1 year experience as Level 2
- Management training completion
- Audit and compliance certification

**Privileges:**
- Supervise multiple kasir
- Approve tutup kasir with large selisih
- Access all system functions
- Conduct training and assessment

### Recertification

**Annual Requirements:**
- Refresher training (4 hours)
- Updated assessment test
- Performance review
- System updates training

**Triggers for Immediate Recertification:**
- 3+ critical errors in 1 month
- Major system updates
- Compliance violations
- Extended absence (>3 months)
