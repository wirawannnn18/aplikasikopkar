Test Task 5.2: Member Segmentation Accuracy Property Test

Testing Property 8: Member Segmentation Consistency - Validates Requirements 3.3

Property-Based Testing Overview

This test validates that member segmentation by transaction volume and frequency is mathematically accurate and consistent across all possible inputs.

Property 8: Member Segmentation Consistency
For any set of members and transactions, the segmentation algorithm must:
  • Categorize every member exactly once
  • Apply threshold rules consistently
  • Maintain mathematical accuracy in calculations
  • Produce deterministic results

Segmentation Thresholds

Segment Min Transactions Min Value (IDR) Description
High Activity ≥ 10 ≥ 1,000,000 Most active members with high transaction volume
Medium Activity ≥ 5 ≥ 500,000 Moderately active members
Low Activity ≥ 1 ≥ 100,000 Members with minimal activity
Inactive < 1 < 100,000 Members below minimum activity threshold
