Manual Testing - Barcode Scanner Pembelian

Informasi: File ini berisi test scenarios untuk fitur barcode scanner pada input pembelian. Buka aplikasi di http://localhost:3000 untuk melakukan testing.

Test Data

Gunakan data barang berikut untuk testing. Pastikan data ini sudah ada di Master Barang:

Sample Barang untuk Testing:
Barcode Nama Barang HPP Stok
1234567890123 Test Item 1 Rp 10.000 100
0001234567890 Test Item 2 (Leading Zeros) Rp 20.000 50
ABC123 Test Item 3 (Alphanumeric) Rp 15.000 75
00000001 Test Item 4 (Many Zeros) Rp 5.000 200
Setup: Jika data belum ada, buat terlebih dahulu di menu Master Barang dengan barcode di atas.

Test Case 1: Basic Barcode Scan Workflow

Scan Barcode Valid

Objective: Memverifikasi bahwa barcode valid dapat di-scan dan auto-fill form

Steps:

  1. Buka aplikasi dan login
  2. Navigasi ke menu "Pembelian"
  3. Klik tombol "Input Pembelian"
  4. Verifikasi focus otomatis di field "Scan Barcode"
  5. Ketik barcode: 1234567890123
  6. Tekan Enter
Expected Result:
  • Dropdown "Pilih Barang Manual" terisi dengan "Test Item 1"
  • Field "Harga Beli" terisi dengan "10000"
  • Focus pindah ke field "Qty"
  • Badge berubah menjadi "✓ Test Item 1" dengan warna biru
  • Badge kembali ke "Scanner Ready" setelah 2 detik
  • Field barcode dikosongkan
Complete Add Item Workflow

Objective: Memverifikasi workflow lengkap dari scan sampai item ditambahkan

Steps:

  1. Lanjutkan dari test case sebelumnya (focus di Qty)
  2. Ketik qty: 5
  3. Tekan Enter
Expected Result:
  • Item "Test Item 1" muncul di tabel dengan qty 5
  • Subtotal = 5 × 10.000 = Rp 50.000
  • Notifikasi sukses: "✓ 'Test Item 1' ditambahkan ke daftar"
  • Field barcode, qty, dan harga dikosongkan
  • Focus kembali ke field barcode
  • Qty field reset ke nilai 1

Test Case 2: Error Handling

Barcode Not Found

Objective: Memverifikasi error handling untuk barcode yang tidak ada

Steps:

  1. Di field barcode, ketik: INVALID999
  2. Tekan Enter
Expected Result:
  • Notifikasi warning: "Barang dengan barcode 'INVALID999' tidak ditemukan"
  • Field barcode dikosongkan
  • Focus tetap di field barcode
  • Dropdown dan harga tidak berubah
Empty Barcode Input

Objective: Memverifikasi handling untuk input kosong

Steps:

  1. Di field barcode (kosong), tekan Enter
Expected Result:
  • Tidak ada notifikasi error
  • Tidak ada perubahan pada form
  • Focus tetap di field barcode
  • Silent ignore (tidak mengganggu user)

Test Case 3: Duplicate Barcode Handling

Scan Same Barcode Twice

Objective: Memverifikasi bahwa scan barcode yang sama akan increment qty

Steps:

  1. Scan barcode: ABC123 + Enter
  2. Input qty: 3 + Enter
  3. Scan barcode yang sama lagi: ABC123 + Enter
  4. Input qty: 2 + Enter
Expected Result:
  • Hanya ada 1 baris "Test Item 3" di tabel
  • Qty = 5 (3 + 2)
  • Subtotal = 5 × 15.000 = Rp 75.000
  • Notifikasi kedua: "✓ Qty untuk 'Test Item 3' ditambahkan (3 → 5)"
  • Tidak ada item duplicate

Test Case 4: Mixed Input Methods

Barcode + Manual Dropdown

Objective: Memverifikasi bahwa barcode dan dropdown dapat digunakan bergantian

Steps:

  1. Scan barcode: 1234567890123 + Enter
  2. Qty: 2 + Enter
  3. Gunakan dropdown manual, pilih "Test Item 2"
  4. Qty: 3
  5. Klik tombol "Tambah"
Expected Result:
  • Tabel berisi 2 item berbeda
  • Item 1: Test Item 1, qty 2
  • Item 2: Test Item 2, qty 3
  • Kedua metode input berfungsi independen
  • Tidak ada konflik data

Test Case 5: Keyboard Navigation

Tab Order

Objective: Memverifikasi urutan tab yang logis

Steps:

  1. Buka modal "Input Pembelian"
  2. Tekan Tab berulang kali
  3. Perhatikan urutan focus
Expected Tab Order:
  • Tab 1: Barcode Input
  • Tab 2: Qty
  • Tab 3: Harga Beli
  • Tab 4: Tombol Tambah
  • Tab 5: Dropdown Pilih Barang
  • Urutan logis dan mengikuti workflow
Enter Key Shortcuts

Objective: Memverifikasi Enter key berfungsi di barcode dan qty field

Steps:

  1. Di barcode field, ketik barcode + Enter (harus trigger search)
  2. Di qty field, ketik qty + Enter (harus add item)
Expected Result:
  • Enter di barcode → Trigger search & auto-fill
  • Enter di qty → Add item ke list
  • Tidak perlu klik mouse
  • Keyboard-only operation berfungsi sempurna

Test Case 6: Edge Cases

Leading Zeros Preservation

Objective: Memverifikasi leading zeros tidak hilang

Steps:

  1. Scan barcode: 0001234567890 + Enter
Expected Result:
  • Barang "Test Item 2" ditemukan
  • Leading zeros tidak dihilangkan
  • Barcode "0001234567890" ≠ "1234567890"
Whitespace Handling

Objective: Memverifikasi whitespace di-trim dengan benar

Steps:

  1. Ketik barcode dengan spasi: ABC123 + Enter
Expected Result:
  • Barang "Test Item 3" ditemukan
  • Whitespace di-trim otomatis
  • Pencarian tetap berhasil

Test Case 7: Visual Feedback

Badge Indicator

Objective: Memverifikasi badge berubah saat scan berhasil

Steps:

  1. Perhatikan badge awal: "Scanner Ready" (hijau)
  2. Scan barcode valid + Enter
  3. Tunggu 2 detik
Expected Result:
  • Badge berubah ke "✓ [Nama Barang]" (biru)
  • Setelah 2 detik, kembali ke "Scanner Ready" (hijau)
  • Animasi smooth
  • User mendapat feedback visual yang jelas

Test Summary

Checklist lengkap untuk memastikan semua fitur berfungsi:

Core Functionality
  • Barcode input field tersedia
  • Auto-focus ke barcode saat modal dibuka
  • Barcode valid → Auto-fill form
  • Barcode invalid → Error notification
  • Empty barcode → Silent ignore
  • Enter key di barcode → Trigger search
  • Enter key di qty → Add item
  • Focus management bekerja dengan baik
Advanced Features
  • Duplicate barcode → Increment qty
  • Subtotal auto-recalculate
  • Notifikasi berbeda (new vs duplicate)
  • Dropdown manual tetap berfungsi
  • Tab order logis
  • Keyboard-only operation
  • Leading zeros preserved
  • Whitespace trimmed
  • Badge indicator berfungsi
  • No memory leaks
